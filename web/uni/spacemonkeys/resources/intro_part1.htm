ArgoUML issues

We decided to use ArgoUML to create our UML diagrams since it was free and cross-platform, meaning all team members could use it. Unfortunately it turns out that ArgoUML won't let you specify arrays of things as a type of an attribute or operation in a class diagram (and yes, we did RTFM, but that part hadn't been written yet!). Therefore some of the types shown in our UML diagrams are not quite correct. We have included the correct types in the notes on the diagram wherever necessary.

Please excuse this slightly messy work-around!
