Style

To make it easier for us to understand one another's Java code we should adhere to some basic style guidelines. These guidelines will also ensure that any javadoc generated pages from our code will look consistant.

  1. Always put javadoc comments before class declarations and all methods in your class. These comments should contain a brief description of what the class or method does and, if necessary, how it is to be used. In the case of class comments you should also include your name, so we know how made what class!
    Here's an example of what it should look like (The javadoc comments are italicised):

    /**

    Example class written by James Nash.

    This class does fuck all - it's just to show you how
    to use the javadoc comments!

    */
    public class Example{

        // you don't need javadoc comments
        // before private data
        private float floater;
        private Object ofDesire;


        /**
         Constructor method.

         This one takes lots of pointless arguments and
         normally this is where I'd explain what they are
         for unless it was really trivial (like in get and
         set methods)
        */
        public Example(String name, Object whatever, float number){

            // some code here

        }

        /**
         method that does fuck all!
        */

        public void doFuckAll(){
            
            // some more code

        }

    }

    Btw, this code would produce these javadocs!

  2. We will also stick to the recommended Java conventions for capitalisation...

    • Class names must always begin with a capital letter. Multiple-word names should use mixed capitalisation. Here are some examples:

      class Example2{}

      class SevenOfNine{}

      // but not...
      class wrongName{}

    • Method and variable names should always begin lowercase. Multiple-word names shoud use mixed capitalisation too. Examples:

      public int number;
      private Object recipientOfSeriousPhysicalAbuse;

      public void getBritneysBrain(){}

  3. All classes must be declared public or private. For some reason javadoc doesn't like them otherwise, so we might aswell.

  4. Although it's not strictly a style guideline remember to always comment your code as you write it! It can be tedious at times but it is definitely better than going through a huge chunk of code you just wrote and trying to figure it out again in order to add comments.
    A good idea is to add short one or two line comments before java statements explaining what they will do.
