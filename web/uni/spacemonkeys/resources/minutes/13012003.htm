Last meeting's minutes

Review the last meeting's minutes in order to clarify any outstanding issues or concerns. Two concerns were raised regarding the fact that we still didn't have a full specification for the project and we were lacking an algorithm to draw a map of the network accurately. Both were to be resolved as soon as possible, preferably before the next meeting.

Focus on Part A

Members present came up with a list of documents that would need to be produced for Part A of the assignment. These included 1.)Project Plan: this document was needed to coordinate and complete various sections of the assignment to a deadline other that the one given by the DCS. The aim was to distribute the work in an efficient manner and have the sections completed before the deadline of Friday 31st January in order to give members of the group sufficient time to prepare all the documents for handing in.

2.)High Level Analysis: this document is a requirement of Part A and is to be discussed and completed by all members of the group.

3.)Requirements Analysis: this document is to discussed and completed by the whole group in a manner that facilitates easy transference to UML notation. The requirements are to be sketched on paper before hand and then transfered to UML by one person.

4.)Test Cases: this document signifies the available test methods that can be used to test the software program. To be completed by at least one member of the group.

Other Business

It was noted that now a project plan had been drawn up, regular meeting would need to take place in order to make sure that sections of the project were being completed on time. If possible these meetings should take place at least twice a week.
