Eventually we will need to find an algorithm that, given a graph as it's input, can create a graphical representation thereof that looks vaguely ok! :)

Ben sent in this suggestion. It hasn't been tested yet, but looks very promising:

  1. Let n = the number of stations that have been given.
  2. Make an 'n by 4' matrix.
  3. Give each station a number from 1 to n, Let s = station number.
  4. Let point (a,b) be the centre of the area that the map will be in, and let r = the minimum distance from (a,b) to an edge.
  5. Let T = 360/n
  6. Let x (for each station) = a + r sin (sT) and y = b + r cos (sT).
  7. In matrix, let each line of 4 values be [s, station acronym, x, y]. In this way you have co-ordinates for each station, from which you can draw the nodes, and lines between each station as needed.
